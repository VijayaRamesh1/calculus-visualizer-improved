Accessibility Statement

Our Commitment to Accessibility

Calculus Visualizer is committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone, and applying the relevant accessibility standards.

Conformance Status

The Web Content Accessibility Guidelines (WCAG) defines requirements for designers and developers to improve accessibility for people with disabilities. It defines three levels of conformance: Level A, Level AA, and Level AAA.

Calculus Visualizer is partially conformant with WCAG 2.1 level AA. Partially conformant means that some parts of the content do not fully conform to the accessibility standard.

Accessibility Features

Calculus Visualizer includes the following accessibility features:

  • Keyboard Navigation: All functionality is accessible through keyboard navigation, with visible focus indicators.
  • Screen Reader Support: We've implemented ARIA attributes and proper semantic HTML to support screen reader users.
  • High Contrast Mode: Our interface respects high contrast settings and provides sufficient color contrast.
  • Text Resizing: Text can be resized up to 200% without loss of content or functionality.
  • Alternative Text: All non-decorative images have descriptive alternative text.
  • Reduced Motion: Animations are disabled for users who prefer reduced motion.
  • Dark Mode: Dark theme option to reduce eye strain.
  • Focus Mode: Distraction-free environment for better concentration.
  • Keyboard Shortcuts: Essential functions are accessible via keyboard shortcuts.

Known Limitations

Despite our efforts to ensure accessibility, there are some known limitations:

  • 3D visualizations may be difficult to comprehend for users with visual impairments. We've provided alternative text descriptions and data tables where possible.
  • Complex mathematical formulas may present challenges for screen reader users. We've tried to provide alternative text and simpler explanations.
  • Interactive simulations rely on visual feedback which may be challenging for some users. We're working on auditory and haptic feedback alternatives.

Accessibility Help

If you encounter accessibility barriers on Calculus Visualizer, please let us know:

  • Email: accessibility@calculusvisualizer.edu
  • Feedback Form: Contact Form

We welcome your feedback on the accessibility of Calculus Visualizer. We're committed to providing an inclusive experience and will make every effort to fix issues and implement improvements.

Accessibility Techniques Used

We've implemented the following techniques to enhance accessibility:

  • Semantic HTML structure with appropriate headings, landmarks, and ARIA attributes
  • Skip navigation links for keyboard users
  • Visible focus indicators for keyboard navigation
  • Color combinations with sufficient contrast ratios (minimum 4.5:1 for normal text)
  • Text alternatives for non-text content
  • Live regions for dynamic content updates
  • Proper form labels and instructions
  • Keyboard-accessible controls
  • Responsive design for different devices and screen sizes
  • Support for text resizing and zoom
  • Reduced motion option
